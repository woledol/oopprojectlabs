# Проект по ООП: определение имени по отчеству

Леонов Алексей 466498

Система состоит из двух сервисов:

1. `user_service` — сайт, регистрация, вход, сессии, история запросов.
2. `predictor_service` — определение имени, от которого образовано отчество.

## Участники проекта

- Алексей Леонов, ИСУ 466498
- Виктор Сидоркин, ИСУ 489709

Важно: по отчеству нельзя надежно узнать имя самого человека. Поэтому система определяет имя родителя, от которого образовано отчество: `Иванович -> Иван`, `Сергеевна -> Сергей`.

## Как запустить

```bash
./run_system.sh
```

После запуска:

- сайт: http://localhost:8000
- Swagger: http://localhost:8000/swagger
- проверка сервиса пользователей: http://localhost:8000/health
- проверка сервиса определения: http://localhost:8001/health

Остановка:

```bash
./stop_system.sh
```

## Быстрая проверка

## Основной сценарий

1. Пользователь регистрируется на сайте.
2. Пользователь вводит отчество.
3. `user_service` проверяет сессию и отправляет запрос в `predictor_service`.
4. `predictor_service` применяет словарь и правила окончаний.
5. Пользователь видит найденное имя, уверенность и причину.
6. Запрос сохраняется в истории пользователя.

## ООП в проекте

- Инкапсуляция: логика регистрации, хранения, проверки пароля и определения имени разнесена по отдельным классам.
- Наследование и абстракции: `UserRepository`, `SessionRepository`, `PredictionClient`, `PatronymicRule`.
- Полиморфизм: разные правила определения имени реализуют общий интерфейс `PatronymicRule`.
- Композиция: сервисы приложения собираются из репозиториев, клиентов и доменных правил.
